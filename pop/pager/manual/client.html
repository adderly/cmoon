为你的站点加上聊天客户端

简介： 本节主要描述为你的站点网页中添加在线客服聊天客户端的方法，以及如何定制属于自己的聊天客户端。
对象： 本节内容针对希望使用在线客服的网站管理员。
预备知识： 添加和定制在线客服客户端，你需要了解基本的 HTML 语法， 和简单的 js 对象使用方法。

完成本站注册后， 在你的网站页面上加入以下代码(页面底部， </body>标签之前)， 便可为你的页面加上聊天客户端。

    <script type="text/javascript" src="http://js.kaiwuonline.com/b/kol.js" ></script>
    <script type="text/javascript">
    bmoon.kol.onready({
        aname: 'MySiteName',
        ...
        }
    });
    </script>

其中， aname 为你注册时候填写的站点名。 聊天客户端的所有行为都可由初始化时的参数来控制， 接下来， 我们来详细了解这些参数(以上代码中的省略号)。

控制聊天客户端的摆放位置

首先， 你需要确定将聊天客户端放在你页面的哪个地方， 如下图所示：

红色框框表示聊天客户端， 我们可以通过 pos 参数来控制其在网页中摆放的位置：

    pos : {
        top: 'VALUE',
        bottom: 'VALUE',
        left: 'VALUE',
        right: 'VALUE'
    }

VALUE 取值可以是数字后面接px、 或者%。 比如： top: '0', left: '35px', bottom: '20%' 都是合法值， 分别代表紧贴页顶， 距左边35px, 离底边整个页高的 20%。

值得注意的是， top, left 的优先级要高于 right, bottom。 比如传 pos: {top: '0', bottom: '20%', left: '35px'} 时，将会把客户端放在紧贴页顶， 距左边35px, bottom 参数将被忽略。

另外， pos 参数的默认值是 {right: '0', bottom: '0'}， 将其放在页面右下角。

深入了解聊天客户端

一般情况下， 聊天客户端有以下两种状态：

mid 为多数客户端初始化时的默认展示界面， 多数是“在线客服”几个字， 告诉访客， 这里可以与网站客户聊天， 有的还配上个美女头像， 那就更诱人了：D

max 为访客触发聊天后的界面，里面有如下几个元素：

  • 2： 提示信息， 告诉访客， 他当前正在与哪位客服聊天， 或者没有客服在线。
  • 3： 关闭聊天窗口， 切回到 mid 状态的控件。
  • 4： 聊天消息列表。
  • 5： 访客输入聊天消息控件。

下面， 我们看看控制这两个状态的一些具体参数。

  • maxevent

    • 由 mid 状态切换到 max 状态的触发方式。 可取值 'click'、 或者'mouseenter'。 比如指定 maxevent: 'mouseenter' 时， 鼠标移入 mid, 将切换到 max 状态， 不用点击。 默认取值为 'click'。

  • defaultUI

    • 默认展现给访客的界面。 可取值 'mid'、 'max'。 默认是展现 mid 界面给访客。

  • restoreUI

    • 在访客刷新页面，或新开标签页时， 是否记住聊天客户端当前的展开状态。 可取值 true、 false。 默认是true， 这样，用户在触发聊天窗口后刷新页面，聊天窗口还是在触发状态(max状态)。

  • restoreMsg

    • 在访客刷新页面，或新开标签页时， 是否展示该访客和客服的最近几条聊天消息给访客。 可取值 true、 false。 默认是true， 这样，用户在刷新页面，或者第二天再打开你的网站时，将能在聊天消息列表中看到于你的最近几次对话。 有情提醒： 隐私性质的网站建议将该值设成 false， 不然客户投诉到时候可不要来找我；D

  • hidemidOnMax

    • 界面由 mid 状态 切换到 max 状态时， 是否隐藏 mid 状态， 默认隐藏， 但有时为了保证界面的一致性， 你可能不希望 mid 被隐藏， 将其设置成 true 就可以了。

再深入一点

好吧， 你一定会奇怪， 为何是 mid 和 max？ 这是因为你可能还需要一个状态： min， 代表聊天客户端最小化时的状态， 如下图：

虽然比较直观， 还是解释下上面的几个数字

  • 1： 最小化 mid 界面， 我们取名叫 miner。
  • 2： 最小化 max 界面， 功能同 miner 一样， 所以也叫 miner。
  • 3： 聊天客户端最小化时的状态， 取名 min。

将 minable 参数设成 true， 便可以使用上面这种带 min 状态的聊天客户端， 很简单吧， 别高兴太早， 因为界面上多了以上三个个控件， 所以客户端的样式文件同样也会复杂一点(我们为此提供了一个默认的 样式文件 )。 编写和指定样式文件的方法将在下节介绍。

定制客户端

不满意我们提供的聊天客户端？ 你可以自己设计各个窗口、空间的大小、风格。。。 设计好后，做好样式文件(.css)文件， 然后通过 css 参数来初始化。 该参数的默认取值是 'http://css.kaiwuonline.com/b/client/lcs.css'。 具体 css 文件的编写方法可以参考：

  • mic ： 只有 mid 和 max 状态的样式。
  • lcs ： 带 min 状态的样式。

各个元素的说明， 在以上示例文件中有简要的解释。 其实很简单， 你需要牢记的仅仅只是 min, mid, max; miner, mider, maxer 代表的意思： min, mid, max 分别代表聊天客户端的最小化， 普通， 最大化 三个状态； miner, mider, maxer 分别代表切换到这三个状态的控件。

小结

好的，一切都快结束了！ 来回顾一下所有的参数：

参数名 类型 是否必填 简介 默认值
aname 字符串 必填 注册时的站点名 必填，无默认值
pos 对象 选填 聊天客户端在页面中摆放的位置 {right: 0, bottom: 0}
maxevent 字符串 选填 窗口由 mid 切换到 max 状态的方式， 可填 click、mouseenter 'click'
defaultUI 字符串 选填 访客来访时， 聊天客户端默认状态， 可取 min, mid, 或max 'mid'
restoreUI 布尔 选填 页面刷新时，是否记住聊天客户端状态 true
restoreMsg 布尔 选填 页面刷新时，是否展示最近几条聊天记录 true
hidemidOnMax 布尔值 选填 窗口由 mid 切换到 max 状态时， 是否隐藏 mid 状态 true
minable 布尔 选填 聊天客户端是否具有 min 状态 false
css 字符串 选填 聊天客户端的样式文件 'http://css.kaiwuonline.com/b/client/lcs.css'
